About

I'm an electrical and computer engineering student at NYU Tandon, and I've never been satisfied knowing that something works without knowing why. That itch pulled me from writing programs to questioning what actually runs them, so I built my own answer: an 8-bit CPU in Verilog with a cache hierarchy and virtual memory, running on real hardware.

The pattern repeats everywhere I work. At Generex I write firmware that has to be trusted before the board is ever powered on. At Tokonoma I build infrastructure that lets AI agents investigate production incidents. Each layer I learn makes the one below it more interesting, and the bottom of that stack is chip design. That's where I'm headed.

Project datasheet — digital design Verilog · FPGA

8-bit CPU with Virtual Memory & Cache

A complete 8-bit Harvard-architecture CPU designed in Verilog: a 12-instruction ISA, ALU, register file, control unit, and a write-back L1 plus 4-way set-associative L2 cache. Extended with full virtual memory through a 4-entry TLB and a cache-mediated MMU with hardware page-table walks and page-fault handling. All 13 modules are verified in simulation and running on a Basys 3 (Artix-7) FPGA, assembled with a custom C++ assembler. Physical tapeout is next.
